# 磁盘清理（当前目录）

扫描当前工作目录，找出占用空间大的文件和可清理的缓存，释放磁盘空间。

## 执行步骤

1. 先确认当前工作目录路径，检查所在磁盘剩余空间
2. 扫描当前目录下的空间占用：
   - 列出当前目录总大小
   - 列出一级子目录/文件大小，按从大到小排序（取 top 15）
   - 识别常见可清理项（node_modules、.cache、__pycache__、.tmp、*.log、dist、build、.next、.nuxt 等）
3. 如果发现大文件（>100MB），单独列出
4. 汇总可清理项目及预计释放空间
5. 向用户确认要清理哪些项目
6. 执行清理，报告每项清理结果
7. 输出清理前后空间对比

## 注意事项

- 每项清理前都要先确认，不要静默删除
- **不要删除** .git 目录、源代码文件、配置文件
- 对于 node_modules，提示用户可以用 `npm install` 重新安装
- 如果当前目录是项目根目录，重点关注构建产物和依赖缓存
